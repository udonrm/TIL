# システム開発の流れ

- 用件定義

  ```
  システム化する範囲と機能を整理
  機能要件：現場からの要望を含めた条件
  非機能要件：開発側が提示する条件、表に出ない

  外部設計：利用者から見た設計（システムの見た目）
  内部設計：開発者から見た設計（DB設計とか開発者視点）
  詳細設計：プログラマへの指示内容となる設計
  ```

